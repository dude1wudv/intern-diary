# 架构文档：实习成果生成器

## 当前事实

- Android：Kotlin、AndroidX、Material Components、OkHttp、ViewBinding。
- 后端：FastAPI、Pydantic、httpx、python-docx、Uvicorn。
- 存储：文件目录，按 `data/workdays/YYYY-MM-DD/` 保存原始文本、整理稿、图片、草稿、Word。
- 鉴权：Bearer Token；可通过 `users.json` 映射 workspace/profile。
- LLM：OpenAI-compatible Chat Completions。

## 目标架构

```text
Android App / Web Console
        |
        v
FastAPI Report API
        |
        +-- Source Aggregator: 按日期范围读取 raw/sorted/image/draft
        +-- Template Registry: 读取内置/用户模板元数据
        +-- LLM Report Generator: 周报/月报/总结结构化生成
        +-- Word Renderer: Markdown/结构化段落 + 模板 -> docx
        +-- Validation: 变量、日期、字数、空内容校验
        |
        v
File Storage: data[/users/<workspace>]/reports/<report_id>/
```

## 技术决策

| 决策 | 选择 | 备选 | 理由 |
|---|---|---|---|
| 报告存储 | 文件系统 `reports/<report_id>` | SQLite/Postgres | 当前项目已文件存储；A 主线无需 DB。 |
| 模板配置 | `templates/report_templates.json` + docx 文件 | 管理后台上传 | 第一版够用，后续再做上传。 |
| 报告生成 | 同步 HTTP 先做 | 后台队列 | 生成量小；已有接口也是同步；超时再队列化。 |
| 导出格式 | Word `.docx` | PDF/Markdown ZIP | 项目核心就是 Word；Markdown 作为中间产物保留。 |
| Android UI | 复用现有 Activity + Sheet/Dialog | 新 Activity / Compose 重构 | 最小改动，避免 UI 大重构。 |
| Subagent | 按文件边界并行 | 多 agent 同改同一文件 | 减少冲突，主线程集成。 |

## 数据结构草案

```json
{
  "report_id": "weekly-2026-07-01_2026-07-07-<shortid>",
  "type": "weekly|monthly|internship_summary",
  "start_date": "YYYY-MM-DD",
  "end_date": "YYYY-MM-DD",
  "template_id": "weekly-default",
  "status": "drafted|failed",
  "markdown_file": "report.md",
  "docx_file": "report.docx",
  "created_at": "ISO-8601",
  "validation": []
}
```

## 数据流

1. 用户选择报告类型、日期范围、模板。
2. 后端校验日期范围和模板存在。
3. Source Aggregator 读取多天素材，拼成 LLM 上下文。
4. LLM 返回结构化内容或 Markdown。
5. Word Renderer 套模板生成 `.docx`。
6. Validation 写入校验结果。
7. API 返回 `report_id`，前端/Android 下载。
